### Kadane's Algorithm
**১. Subarray কী? (What is a Subarray?)**
একটি অ্যারের কিছু অংশ যদি ক্রমানুসারে (order ঠিক রেখে) নেওয়া হয়, তবে তাকে Subarray বলে। এটি মূল অ্যারের একটি ছোট টুকরো।

**উদাহরণ:**
ধরি আমাদের অ্যারেটি হলো: [A, B, C, D]
এর সম্ভাব্য কিছু Subarray হলো:
- [A]
- [A, B]
- [B, C, D]
- [A, B, C, D] (পুরো অ্যারেটিও একটি সাব-অ্যারে)

**২. Contiguous বা Continuous বলতে কী বোঝানো হয়েছে?**
Kadane's Algorithm-এর ক্ষেত্রে Contiguous কথাটি সবচেয়ে গুরুত্বপূর্ণ। এর মানে হলো: আপনি অ্যারের মাঝখান থেকে কোনো সংখ্যা বাদ দিতে পারবেন না। সংখ্যাগুলো অবশ্যই গায়ে গায়ে লাগানো বা পাশাপাশি থাকতে হবে।

সহজ উদাহরণ দিয়ে বুঝুন:
অ্যারে: [10, -5, 20, 30]

- Contiguous (Subarray): [10, -5, 20] — এটি একটি সাব-অ্যারে কারণ সব উপাদান সিরিয়াল অনুযায়ী আছে।
- Not Contiguous (Subsequence): [10, 20, 30] — এটি সাব-অ্যারে নয়, কারণ আপনি মাঝখান থেকে -5 কে লাফ দিয়ে পার করে গেছেন। এটাকে বলা হয় Subsequence।

**৩. Kadane’s Algorithm Theory: গভীরে বুঝুন**
Kadane’s Algorithm মূলত একটি Greedy/Dynamic Programming পদ্ধতি। এটি প্রতিটা স্টেপে একটি সিদ্ধান্ত নেয়।

কল্পনা করুন আপনি রাস্তা দিয়ে হাঁটছেন এবং পথে টাকা (+ নম্বর) অথবা গর্ত/খরচ (- নম্বর) পাচ্ছেন। আপনার লক্ষ্য হলো সবচেয়ে বেশি টাকা জমানো।

অ্যালগরিদমটি যেভাবে কাজ করে:
* ১. আপনি যখন নতুন একটি ঘরে (Element) পৌঁছান, আপনার কাছে দুটি অপশন থাকে:
- অপশন এ: আগের জমানো টাকার সাথে বর্তমান ঘরের টাকা যোগ করা।
- অপশন বি: যদি আগের জমানো টাকা মাইনাস হয়ে আপনার বর্তমান অবস্থাকে আরও খারাপ করে দেয়, তবে আগের সব ছেড়ে দিয়ে এই ঘর থেকেই নতুন করে জমা শুরু করা।

- ২. এই দুটি অপশনের মধ্যে যেটি বড় (Math.max), সেটিই হবে আপনার সেই মুহূর্তের currentMax।
- ৩. পুরো পথে আপনার হাতে সর্বোচ্চ যত টাকা উঠেছিল, সেটাই হলো globalMax বা maxSoFar।
